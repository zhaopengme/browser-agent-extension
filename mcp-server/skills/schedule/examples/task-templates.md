# 常见任务模板

## 代码质量

### 每日代码检查
```
名称: daily-code-check
频率: 每天早上 9 点
Cron: 0 9 * * *
操作: 运行 lint、format、test
```

### 依赖更新检查
```
名称: check-dependencies
频率: 每周一上午 10 点
Cron: 0 10 * * 1
操作: npm outdated / yarn upgrade-interactive
```

## 文档

### 每周生成报告
```
名称: weekly-report
频率: 每周五下午 5 点
Cron: 0 17 * * 5
操作: 生成项目状态报告
```

### API 文档同步
```
名称: sync-api-docs
频率: 代码变更后触发
Cron: 手动触发
操作: 从代码注释生成 API 文档
```

## 备份

### 每日数据备份
```
名称: daily-backup
频率: 每天凌晨 2 点
Cron: 0 2 * * *
操作: 备份数据库/重要文件
```

### 每周快照
```
名称: weekly-snapshot
频率: 每周日凌晨 3 点
Cron: 0 3 * * 0
操作: 创建项目快照
```

## 监控

### 健康检查
```
名称: health-check
频率: 每 30 分钟
Cron: */30 * * * *
操作: 检查服务状态
```

### 磁盘空间检查
```
名称: disk-space-check
频率: 每天上午 8 点
Cron: 0 8 * * *
操作: 检查磁盘使用率，超过阈值告警
```

## 通知

### 每日提醒
```
名称: daily-reminder
频率: 每天早上 9 点
Cron: 0 9 * * *
操作: 发送今日待办事项
```

### 每周总结
```
名称: weekly-summary
频率: 每周日晚上 8 点
Cron: 0 20 * * 0
操作: 发送本周工作总结
```

## 自定义模板

创建任务时可以参考这些模板，调整：
- 频率（cron 表达式）
- 具体操作（命令或 MCP 调用）
- 通知方式（成功/失败时通知）
